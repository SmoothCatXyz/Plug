02

Chrome Browser Relay — 差异化武器

核心优势：其他工具用的是"干净 Chromium"（无登录态），Alma 通过 Chrome Extension 接管用户真实 Chrome，携带完整 cookies/session，可以操作飞书、内网 Jira、公司邮箱等需要登录的系统。

架构链路

1
AI 决策调用 browser_* 工具
2
Alma 主进程发送指令 → WebSocket (localhost)
3
Chrome Extension 接收，使用 chrome.debugger API
4
CDP (Chrome DevTools Protocol) 控制目标 Tab
5
操作结果（截图/文本/状态）回传 Alma 注入 context

支持操作

导航 Navigate
访问 URL，等待页面加载完成
截图 Screenshot
捕获当前 Tab 视口，返回图像给视觉模型
交互 Click / Type
点击元素、填写表单，可触发登录后的工作流
文本提取 Get Text
提取 DOM 文本内容，用于分析/总结
Source: alma.now/docs/features/chrome-relay
03

记忆系统 Memory System

🧠
双模型架构
主 LLM（用户选择的对话模型）负责理解和生成。

独立 Embedding Model（默认 text-embedding-3-small）专门做记忆向量化和语义检索，不占用主模型 context。
text-embedding-3-small local vector db
🔄
自动记忆生命周期
对话结束 → 提取重要事实
向量化 → 存入本地向量库
每次回复前 → 语义相似度检索 → top-k 注入 context
Incognito 模式 → 完全禁用读写，当次会话不留痕
importance score confidence tag 3-layer indexing

记忆召回流程

①
用户发送新消息 → 提取查询向量（embedding model）
②
向量库余弦相似度搜索，返回 top-k 条记忆（按相关度排序）
③
相关记忆自动注入 system prompt 上部（用户不可见）
④
LLM 生成回复时自然融入记忆上下文，用户感受到"AI 记住了我"
⑤
对话结束 → 提取模型识别新事实 → 写入向量库（带 importance/confidence 评分）
vs 纯 Markdown Memory（如当前 Plug MVP）： Markdown 方案随时间膨胀 → token 成本爆炸 + 旧记忆污染上下文。 Alma 的向量检索只注入当前最相关的记忆，成本恒定，召回精度更高。 用户还可以手动查看、编辑、删除具体记忆条目。
Source: alma.now/docs/features/memory
04

Agent 性格设计

公开信息有限：Alma 是闭源产品，system prompt 未公开泄露。以下分析基于产品行为观察和 yetone 的公开表述。
🎭
Agent Crew 多角色
Alma 不是单一固定性格。Agent Crew 允许为每个子 Agent 设定：
• 不同的 Model（Claude vs GPT 有不同风格）
• 不同的 Perspective/Persona
• 不同的可用工具集

复杂任务由 Crew 拆分并分配给最合适的 Agent，性格随功能角色变化。
per-agent persona
🤖
AI-Native 自操作
yetone 原话："Alma 可以按照你的指示自己操作自己"。

AI 可以操作 Alma 的 UI 来帮用户配置 Alma，也可以解释如何设置。

这暗示 Alma 的 system prompt 中包含对自身 UI/功能的详细描述，构建了一种"元认知"性格——AI 知道自己是什么。
meta-agent self-aware
📐
模型微调 (未证实)
中文社区评论称 yetone "fine-tuned all current mainstream models"， 使 Alma 的体验在各模型上趋于一致。

更可能的解释：通过精细 system prompt engineering 和 prompt routing 实现跨模型统一行为，而非真正 fine-tune。
prompt engineering
05

用户关怀设计

Alma 的"关怀感"主要来自技术层面的设计，而不是显式的情感 AI 功能。核心策略是通过记忆连续性制造"AI 认识你"的感受。

已确认的关怀机制

🧠 记忆连续性
跨对话记住用户偏好、工作上下文、历史决策。用户感受到 AI "认识自己"，而不是每次从零开始。
🕵️ Incognito 模式
明确告知用户"这次对话不会被记住"，尊重隐私边界，给用户控制感。
🎤 Whisper 语音输入
语音交互降低沟通门槛，对习惯语音记录的用户更自然、更接近"助理"感。
📱 iPhone Mirroring 支持
支持 iOS 镜像文本输入，用户随时随地都能与 Alma 交互，无摩擦感。

值得注意的缺位

❌ 未见主动关怀
没有公开证据显示 Alma 有主动问候、情感安抚、用户情绪识别等功能。定位是效率工具而非情感 AI。
❌ 无用户状态追踪
没有"你上次在做 X，要继续吗？"的会话续接设计（这在 Copilot/Gemini 中较常见）。
❌ 无显式关系层
不同于 Character.ai 或 Pi.ai，Alma 没有建立"AI 伴侣/导师"角色。偏向工具性，记忆系统是功能增益而非情感纽带。
⚡ 设计哲学差异
Alma 的关怀逻辑：让 AI 更有用（记忆、效率）= 间接关怀。而非显式的情感设计。对效率优先的知识工作者这条路更可持续。

07

结论 & 对 Plug 的启示

Alma 的核心护城河

1. 向量记忆系统：最大的技术壁垒，让 AI 真正"认识"用户，产生粘性
2. Chrome Browser Relay：真实登录态，是其他工具做不到的
3. Agent Crew：多 Agent 协作是复杂任务的关键，Alma 走得最远
4. Plugin 生态：开放平台策略，积累网络效应
5. 发布速度：~4 releases/day，用户反馈 → 修复 → 上线极快

Plug 的差异化空间

1. PM 垂直深度：Alma 不专注 PM，Plug 可以在 PRD/竞品/交付物上做深
2. 机甲座舱体验：差异化的 UI/UX 定位，有辨识度
3. 国产企业工具生态：飞书/语雀/PingCode MCP 是 Alma 尚未覆盖的
⚠️ 最紧迫的 Gap：向量记忆系统是 Plug 必须实现的，否则用户忠诚度会输给 Alma
